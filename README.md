7. [Задача "распределённого консенсуса"](hw7)

**Формулировка**:

33 богатыря (`n процессов`) решили спеть песню (`строка из символов`).

Каждый богатырь выбирает одно слово (`символ из строки`) и будет петь (`выводить на экран`) только его.

В песне могут быть повторяющиеся слова, при этом уникальных слов не больше числа богатырей.

Необходимо написать программу при помощи **сообщений** (можно отправлять как всем процессам, так и выбранному процессу), которая симулирует такое межпроцессное взаимодействие.

**Нельзя**:

1. Как-либо *нумеровать* процессы заранее
2. Делать заранее какой-либо процесс *главным*

**Т.е. у всех процессов одинаковые начальные условия и алгоритм**

**Стоит помнить**:
    У процессов должен быть приоритет на приём сообщений перед отправкой.

Пример того, что может пойти не так:

+ 1024 процесса решили отправить широковещательное сообщение
+ очереди сообщений заполнились (их никто не читает)
+ **deadlock**.
